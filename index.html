Introduction to HTML

HTML stands for HyperText Markup Language. It is the standard language used to create and structure web pages.

What is HyperText?

HyperText refers to text that links to other text (documents, media, etc.). HTML allows this via the <a> tag.

Why Use HTML?

  • It provides the structure for every webpage.
  • HTML is easy to learn and platform-independent.
  • It works with CSS for styling and JavaScript for behavior.

Basic HTML Document Structure

<!DOCTYPE html>
<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>

Key Points

  • HTML uses tags like <h1>, <p>, <a> to format content.
  • Tags usually come in pairs: opening and closing tags.
  • HTML is not a programming language; it’s a markup language.

HTML Version History

Version Year Features
HTML 1.0 1993 Basic tags only
HTML 2.0 1995 Forms, tables
HTML 4.01 1999 More structured formatting
HTML5 2014 Semantic tags, multimedia support, mobile-friendly

HTML Document Structure

An HTML document has a well-defined structure consisting of several key elements. These elements help the browser understand how to render the page.

Basic Structure

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
  </head>
  <body>
    <!-- Page content goes here -->
  </body>
</html>

Explanation of Each Part

  • <!DOCTYPE html> – Declares the document as HTML5.
  • <html> – The root element of the page.
  • lang="en" – Specifies the language of the document (English here).
  • <head> – Contains meta information, styles, scripts, and title.
  • <meta charset="UTF-8"> – Sets character encoding to support all characters.
  • <meta name="viewport"> – Makes the site responsive on all devices.
  • <title> – The title that appears on the browser tab.
  • <body> – Contains all the visible content of the webpage.

Visual Structure

The structure helps browsers parse and display the webpage correctly. Developers and tools also rely on this consistency to interpret the document.

Tips

  • Always include the <!DOCTYPE> at the top.
  • Make sure the <head> has necessary meta tags for SEO and responsiveness.
  • Keep your content inside the <body> tag only.

HTML Headings and Paragraphs

HTML provides a simple way to organize content using headings and paragraphs. These elements help structure the document and improve readability for users and search engines.

HTML Headings

HTML offers six levels of headings, from <h1> (most important) to <h6> (least important). 

<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Section Heading</h3>
<h4>Smaller Heading</h4>
<h5>Even Smaller</h5>
<h6>Smallest Heading</h6>

Use headings to create a clear hierarchy in your page. Do not skip heading levels (e.g., from <h1> to <h4>) unless necessary.

HTML Paragraphs

Paragraphs are defined using the <p> tag. They separate blocks of text for better formatting and flow.

<p>This is a paragraph of text in HTML.</p>

Browsers automatically add space before and after each paragraph. Avoid using multiple <br> tags for spacing—use CSS instead.

Best Practices

  • Use only one <h1> per page, typically for the main title.
  • Use headings in order (e.g., <h2> under <h1>).
  • Keep paragraph text concise and readable.
  • Use CSS for advanced styling and layout of headings and text.

HTML Formatting Tags

HTML provides several formatting tags that help emphasize or style portions of text. These tags do not affect the layout but define the meaning or appearance of content.

Common Formatting Tags

  • <b> – Makes text bold
  • <strong> – Indicates important text (bold by default)
  • <i> – Makes text italic
  • <em> – Emphasizes text (italic by default)
  • <mark> – Highlights text
  • <small> – Displays smaller text
  • <del> – Strikes through text (deleted)
  • <ins> – Underlines inserted text
  • <sub> – Subscript text like H2O
  • <sup> – Superscript text like x2

Example Usage

<p>This is <b>bold</b> and this is <strong>important</strong>.</p>
<p>Use <i>italics</i> or <em>emphasis</em> for stress.</p>
<p>Here is <mark>highlighted</mark> text.</p>
<p>Water formula: H<sub>2</sub>O</p>
<p>Math: x<sup>2</sup></p>

Best Practices

  • Use <strong> and <em> for semantic importance — screen readers recognize them.
  • Use <b> and <i> only for visual styling without meaning.
  • Use CSS if you need advanced text styling and effects.

HTML Links (<a>)

HTML links are used to connect to other documents or sections. The <a> tag defines a hyperlink.

Syntax

<a href=\"https://example.com\">Visit Example</a>

Attributes

  • href – Specifies the destination URL.
  • target – Specifies where to open the link:
    • _self (default): Opens in the same tab
    • _blank: Opens in a new tab
  • title – Tooltip shown on hover.

Example

<a href=\"https://www.kabcoders.com\" target=\"_blank\" title=\"Kabcoders Official\">
  Visit Kabcoders
</a>

Output: Visit Kabcoders

HTML Images (<img>)

The <img> tag is used to embed images in a webpage. It is a self-closing tag and requires at least the src and alt attributes.

Syntax

<img src=\"image.jpg\" alt=\"Description of the image\">

Important Attributes

  • src – Specifies the path to the image file (URL or relative path).
  • alt – Provides alternative text if the image cannot be displayed.
  • width and height – Set the size of the image (in px or %).
  • title – Adds a tooltip that appears when you hover over the image.

Example

<img src=\"https://picsum.photos/200/300" alt=\"Placeholder Image\" width=\"150\" height=\"150\" title=\"Sample\">

HTML Lists (<ul>, <ol>, <li>)

HTML provides different types of lists to organize content. The most common are:

  • Unordered List - <ul>
  • Ordered List - <ol>
  • List Item - <li> (used inside <ul> or <ol>)

Unordered List Example

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Output:

  • HTML
  • CSS
  • JavaScript

Ordered List Example

<ol>
  <li>Step One</li>
  <li>Step Two</li>
  <li>Step Three</li>
</ol>

Output:

  1. Step One
  2. Step Two
  3. Step Three

HTML Tables

HTML tables allow you to arrange data in rows and columns using the <table> element and its child tags.

Basic Tags

  • <table> – Defines the table
  • <tr> – Table row
  • <th> – Table header
  • <td> – Table cell (data)

Basic Table Example

<table>
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>John</td>
    <td>25</td>
  </tr>
  <tr>
    <td>Alice</td>
    <td>30</td>
  </tr>
</table>

Output:

Name Age
John 25
Alice 30

HTML Forms

HTML forms are used to collect user input. The <form> element wraps various input elements like text fields, radio buttons, checkboxes, and submit buttons.

Form Attributes

  • action – The URL where form data is sent.
  • method – HTTP method: GET or POST.

Basic Form Example

<form action=\"/submit\" method=\"post\">
  <label for=\"name\">Name:</label>
  <input type=\"text\" id=\"name\" name=\"name\"><br><br>

  <label for=\"email\">Email:</label>
  <input type=\"email\" id=\"email\" name=\"email\"><br><br>

  <input type=\"submit\" value=\"Submit\">
</form>

HTML Input Elements

The <input> element is one of the most important form elements in HTML. It allows users to enter data in various formats.

Common Input Types

  • text – Single-line text input
  • password – Masks characters
  • email – Validates email format
  • number – Accepts numeric input
  • date – Date picker
  • radio – Single choice from options
  • checkbox – Multiple selections
  • submit – Submits the form

Input Example

<form>
  Name: <input type="text" name="name"><br>
  Password: <input type="password" name="password"><br>
  Email: <input type="email" name="email"><br>
  Age: <input type="number" name="age"><br>
  Birthday: <input type="date" name="dob"><br>

  Gender:
  <input type="radio" name="gender" value="male"> Male
  <input type="radio" name="gender" value="female"> Female<br>

  Hobbies:
  <input type="checkbox" name="hobby" value="reading"> Reading
  <input type="checkbox" name="hobby" value="sports"> Sports<br>

  <input type="submit" value="Submit">
</form>

Output:

Name:
Password:
Email:
Age:
Birthday:

Gender: Male Female

Hobbies: Reading Sports

HTML Semantic Tags

Semantic tags clearly describe their meaning to both the browser and the developer. They improve the readability and SEO of your webpage.

Common Semantic Tags

  • <header> – Defines a page or section header
  • <nav> – Navigation links
  • <main> – Main content area
  • <section> – Thematic grouping of content
  • <article> – Independent self-contained content
  • <aside> – Content aside from the main content (like a sidebar)
  • <footer> – Footer for a document or section

Semantic Structure Example

<header>
  <h1>My Website</h1>
</header>

<nav>
  <a href="#">Home</a>
  <a href="#">About</a>
</nav>

<main>
  <article>
    <h2>Welcome</h2>
    <p>This is my article.</p>
  </article>
</main>

<footer>
  <p>© 2025 My Website</p>
</footer>

Output (structure only, no rendering differences visually unless styled):

HTML Block vs Inline Elements

In HTML, elements are either block-level or inline-level, based on how they behave in the document layout.

Block-Level Elements

  • Start on a new line
  • Take up the full width available
  • Can contain inline and other block elements
  • Examples: <div>, <p>, <h1> to <h6>, <ul>, <table>

Inline-Level Elements

  • Do not start on a new line
  • Take only as much width as necessary
  • Can contain only text or other inline elements
  • Examples: <span>, <a>, <strong>, <em>, <img>

Example Comparison

<div>This is a block-level element.</div>
<span>This is an inline element.</span>

Output:

This is a block-level element.
This is an inline element.

HTML Multimedia (Audio, Video)

HTML allows you to embed multimedia files like audio and video directly into your web pages using the <audio> and <video> elements.

<audio> Element

The <audio> tag is used to embed sound content in documents. It supports controls like play, pause, and volume.

<audio controls>
  <source src="audiofile.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

Output:

<video> Element

The <video> tag is used to embed video content. You can specify width, height, controls, autoplay, loop, etc.

<video width="320" height="240" controls>
  <source src="videofile.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

Output:

HTML Iframes

The <iframe> element is used to embed another HTML page or external content within the current webpage.

Basic Syntax

<iframe src="https://example.com" width="600" height="400">
  Your browser does not support iframes.
</iframe>

Output:

Common Attributes

  • src – URL of the document to embed
  • width and height – Size of the iframe
  • title – Accessibility title for screen readers
  • frameborder – Border width (deprecated, use CSS)
  • allowfullscreen – Allows fullscreen for videos and games

Use Cases

  • Embedding YouTube videos
  • Displaying maps (e.g., Google Maps)
  • Embedding other web pages or tools (like forms or widgets)

HTML Comments

Comments in HTML are not displayed in the browser. They are used to leave notes or explanations within the code for developers.

Syntax

<!-- This is a comment -->

Uses of Comments

  • To explain sections of code
  • To leave reminders or notes
  • To temporarily disable parts of code without deleting them

Example

<!-- This section displays a welcome message -->
<h1>Welcome to My Website</h1>

<!-- <p>This paragraph is temporarily hidden.</p> -->

Comments help maintain clean and understandable code, especially in collaborative projects.

HTML Entities

HTML entities are used to display reserved characters in HTML or characters that do not appear on the keyboard.

Why Use Entities?

  • To display special symbols like <, >, &, ©, etc.
  • To prevent HTML from interpreting certain characters as code

Common Entities

Character Entity Name Entity Code
< &lt; <
> &gt; >
& &amp; &
" &quot; "
© &copy; ©
€ &euro; €

Example

<p>Use &amp; to display an ampersand (&).</p>

Output: Use &amp; to display an ampersand (&).

HTML5 New Elements

HTML5 introduced several new semantic elements that improve the structure and accessibility of web pages.

Key HTML5 Elements

  • <header> – Represents introductory content or navigation links.
  • <nav> – Contains navigation links.
  • <article> – Represents a self-contained piece of content (e.g., blog post).
  • <section> – Represents a section of content with a related theme.
  • <aside> – Contains content tangentially related to the main content (e.g., sidebars).
  • <footer> – Contains footer content like author info or copyright.
  • <figure> and <figcaption> – Used for images with captions.
  • <mark> – Highlights or marks text.
  • <time> – Defines a specific time or date.

Example

<article>
  <header>
    <h2>HTML5 Elements</h2>
  </header>
  <p>This article explains new semantic tags introduced in HTML5.</p>
  <footer>Written by ABCoders</footer>
</article>

These semantic elements improve the readability of your code and help search engines and screen readers understand your content better.

HTML Best Practices

Writing clean, semantic, and well-structured HTML improves website performance, accessibility, and SEO. Here are some best practices every developer should follow:

1. Use Semantic Tags

Use tags like <article>, <section>, <header>, <footer> to describe the content’s purpose.

2. Proper Indentation

Indent your code to improve readability and maintainability.

3. Use Lowercase for Tags and Attributes

HTML is not case-sensitive, but using lowercase is standard and consistent.

4. Quote Attribute Values

Always wrap attribute values in double quotes: <img src="logo.png" alt="Logo">

5. Add ALT Text for Images

This improves accessibility and helps search engines understand the content of images.

6. Validate Your Code

Use the W3C Validator to check your HTML for errors or outdated practices.

7. Keep It DRY (Don't Repeat Yourself)

Use reusable components and avoid duplicating code unnecessarily.

8. Comment Your Code

Use <!-- comments --> to explain sections, especially for complex structures.
